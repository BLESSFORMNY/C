RSA1.c의 개선점
1. pow 함수
- 사용하고 있는 변수의 데이터 타입은 int인 반면, pow 함수는 double일 때 작용
- 그렇다고 double을 데이터 타입으로 사용한다면 소수점 관련하여 문제 발생
2. int 데이터 타입
- 실제 RSA암호는 매우 큰 용량을 다루고 있음
- 데이터 타입을 바꿔야 함
- 일부는 int => long long
3. 코드문 형태
- 소수판별, gcd 같은 것은 따로 함수로 정의하는 것이 나음(굳이)
4. d 관련
- d를 사람이 직접 입력하는 것은 힘듦
- 유클리드 알고리즘을 통한 자동화
